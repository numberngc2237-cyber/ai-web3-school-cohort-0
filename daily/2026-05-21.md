### 关键概念

用自己的话记录的今日核心理解：

1. **LLM = 概率引擎，不是事实数据库**

   LLM（大语言模型）本质上是一个通过海量文本训练出来的概率模型。它生成的每一个词，都是基于前文算出来"最合理接下去的词"——而非从一个数据库中检索事实。
   - Token 是模型处理文本的最小单位（不是"字"），代码和 JSON 比中文更吃 token
   - Embedding 把文本变成数字向量，用于搜索、聚类——搜索能帮你找到相关内容，但不能自己判断正误
   - Transformer + Attention 是 LLM 的核心架构，让模型能在上下文中发现关联模式
   - 幻觉（Hallucination）：模型会自信地输出看起来合理但实际不真实的内容。不能光靠"写好 prompt"解决，要靠外部校验

   **一句话**：LLM 是推理入口，不是最终验证。

2. **Prompt = 你和模型之间的接口协议**

   Prompt 不只是"问一句话"，而是一套完整的任务沟通协议。好的 Prompt 需要拆成四段：任务目标 → 可用输入 → 禁止行为 → 输出格式和失败格式。
   - Few-shot：放少量示例让模型模仿——但容易过时，需要在协议升级后同步更新
   - 结构化输出（Structured Output）：让模型返回固定格式（如 JSON），方便代码检查、校验、记录
   - Prompt Injection（提示注入）：外部内容可能让模型忽略原始规则。防御靠标记不可信数据 + 工具调用前校验 + 高风险动作人工确认

   **一句话**：Prompt 是软约束，真正的安全边界在代码和权限。

3. **Context = 模型能"看见"的信息空间，本质是信息治理问题**

   模型只能基于它能"看见"的内容来行动。系统必须决定：什么信息能进上下文、带着什么身份进去、过期后怎么退出。
   - Context Window：模型一次能处理的信息上限——窗口大了不代表模型会注意到每一个细节
   - Context Engineering：设计信息的进入方式——选择数据源、排序、裁剪、标注来源、隔离不可信内容
   - Memory ≠ 实时授权：用户过去允许≠现在仍然允许
   - 知识库需要标注来源、更新时间、协议版本

   **一句话**：Context 是连接模型和现实系统的入口——信息分层越清楚，越安全可控。

### 🔗 三者关系

```
LLM（理解和生成层）
  ↑ Prompt 约束任务边界
  ↑ Context 提供可信信息

Prompt → 定义任务和输出格式（软约束，不能替代安全边界）
Context → 决定模型能看到什么（信息治理，分层管理）
```

**贯穿三章的核心思想**：模型是推理工具，不是真相之源。越靠近真实执行（链上交易、资产操作），越需要代码校验 + 外部验证 + 人工确认。

### 产品视角

以产品研究方向思考今日所学：

| 概念 | 产品场景 | 用户是谁 | 解决什么问题 |
|------|---------|---------|------------|
| **LLM 能力边界** | AI 客服 / AI 分析师 / AI 交易顾问 | 普通用户期望 AI 给出"正确"答案；产品经理需要设计"能做/不能做"边界 | 用户默认以为 AI"什么都知道"——产品需明确告知能力边界，避免过度信任 |
| **Prompt 设计** | 用户引导界面 / 模板系统 / 对话流设计 | 无技术背景的用户不知道怎么写 Prompt → 产品需预设模板 | 降低使用门槛：把 Prompt 工程变成产品功能（预设场景、引导式输入） |
| **Context 限制** | 长文档处理 / 历史对话管理 / 多轮交互 | 需要处理大量资料的专业用户（律师、研究员、交易员） | Context 决定产品能处理多复杂的任务——超出窗口的内容需检索/摘要策略 |
| **Token 成本** | SaaS 定价 / API 调用优化 | 产品方每次调用都花钱 | Token 消耗直接影响产品成本——需考虑输入格式优化 |
| **Hallucination** | 金融/医疗/法律领域 AI 产品 | 高风险场景用户"出错"代价极高 | 产品必须设计校验层：来源引用、数据校验、人工确认环节 |

**核心产品洞察**：
- AI 产品的信任建立不靠"模型更聪明"，而靠"系统更透明"——让用户看到信息来源、确定性标记、不确定时的替代方案
- 无代码用户接触 AI 的第一界面是 Prompt（输入框/对话），产品要帮用户把模糊意图翻译成精确任务
- Context 窗口不是无限大 → 产品需要设计"什么该记住、什么该忘掉"——对用户是体验问题，对产品是成本和安全问题
