## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ 核心概念。

- 📖 [氛围编程 Vibe Coding](https://aiweb3.school/zh/handbook/ai/vibe-coding/) — 为什么要学 + 第一性原理
- 🎯 重点理解三个问题：(1) 什么是 Vibe Coding？——用自然语言描述需求，AI 生成代码，你不需要写一行代码 (2) 和传统编程的本质区别：重心从「怎么写」移到「要什么」 (3) 适用边界——什么时候 Vibe Coding 够用，什么时候需要专业开发者介入
- 💡 产品思考：如果你能用 Vibe Coding 在 1 小时内做一个可交互的产品 Demo，你的产品验证流程会怎么改变？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读「工具与平台」+「实践模式」+「在 AI × Web3 中的位置」。

- 📖 同上链接 — 工具与平台（Bolt、v0、Lovable、Cursor 等）+ 实践模式 + AI × Web3 中的位置
- 🎯 重点了解：(1) 主流无代码/低代码 AI 开发工具对比 (2) 典型的 Vibe Coding 工作流：描述 → 生成 → 预览 → 迭代反馈 (3) 为什么产品经理学会 Vibe Coding 能极大提升与开发团队的协作效率
- 💡 产品思考：选一个你想做的 AI × Web3 产品，想一想用 Vibe Coding 能快速验证哪些假设？

### 🟣 挑战路径（~2h）

完成推荐路径后，做最小实践 + 阅读扩展资料。

- 📖 同上链接 — 最小实践 + 扩展阅读
- 🔬 最小实践（产品方向，无需代码基础）：
  1. 注册一个 Vibe Coding 工具（推荐 Bolt.new 或 Lovable）
  2. 用自然语言描述一个简单的 AI × Web3 产品页面（如「做一个 AI Agent 聊天界面，用户可以输入问题，Agent 返回回答和链上数据」）
  3. 迭代 3 轮：第一版只管界面 → 第二版加交互 → 第三版加数据展示
  4. 记录：每轮用了多少时间、AI 懂了什么、改了什么
- 📚 扩展阅读：[Bolt.new](https://bolt.new/)、[v0 by Vercel](https://v0.dev/)、[Cursor](https://cursor.sh/)、[Lovable](https://lovable.dev/)
- ✍️ 产品分析：对比至少 2 个 Vibe Coding 工具的定位差异——各适合什么类型的用户？解决什么痛点？
