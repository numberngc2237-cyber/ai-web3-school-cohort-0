## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ Solidity + ABI + Event 三个初级知识节点。

- 📖 [智能合约 Smart Contract](https://aiweb3.school/zh/handbook/web3/smart-contract/) — 为什么要学 + 第一性原理 + Solidity + ABI + Event
- 🎯 重点理解：合约=链上公开规则，一旦部署修改成本极高；ABI 是机器可读接口但"不是安全说明书"；Event 是合约的日志系统
- 💡 产品思考：为什么 DeFi、NFT 的核心逻辑必须写成智能合约而不是后端 API？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读 EVM（中级）+「在 AI × Web3 中的位置」+「常见错误」。

- 📖 同上链接 — EVM + 在 AI × Web3 中的位置 + 常见错误
- 🎯 重点理解：EVM 为什么需要 gas、为什么 storage 贵；AI 做建议和编排 → 钱包做授权 → 合约做可验证执行 → 监控记录结果
- 💡 产品思考：如果 AI Agent 帮你调用合约，哪些步骤必须由用户人工确认？

### 🟣 挑战路径（~2h）

完成推荐路径后，读 Upgrade（高级）+ 做「最小实践」+ 阅读扩展资料。

- 📖 同上链接 — Upgrade（可升级 vs 不可升级、升级风险四问）
- 🔬 最小实践：在区块浏览器找一个已验证的简单 ERC-20 合约 → 读源码 → 找权限函数 → 找事件 → 分析风险边界（无需写代码）
- 📚 扩展阅读：[Solidity Docs](https://docs.soliditylang.org/)，[Ethereum Smart Contracts](https://ethereum.org/en/developers/docs/smart-contracts/)，[OpenZeppelin Contracts](https://www.openzeppelin.com/contracts)
- ✍️ 产品分析草稿：对比 Uniswap V2 / V3 / V4 的合约架构演进——每次升级改变了什么？用户感知到了什么？
