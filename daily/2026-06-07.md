- 📖 [MCP Model Context Protocol](https://aiweb3.school/zh/handbook/ai/mcp/) — 为什么要学 + 第一性原理
- 🎯 重点理解三个问题：(1) 什么是 MCP？——连接 AI 模型与外部工具的开放标准协议 (2) MCP 和传统 API 调用的本质区别：API 是「每次手动对接」，MCP 是「即插即用的 USB 接口」 (3) 为什么产品经理需要关注 MCP？——它定义了 AI 产品的「插件生态」，会影响产品的可扩展性
- 💡 产品思考：如果所有 AI 工具都遵循 MCP 协议，这会如何改变产品的竞争格局？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读「核心架构」+「工具与资源」+「在 AI × Web3 中的位置」。

- 📖 同上链接 — 核心架构（Client-Server 模型）+ 工具与资源（Resources / Tools / Prompts）+ AI × Web3 中的位置
- 🎯 重点了解：(1) MCP 的三层架构：Host → Client → Server (2) 三大原语：Resources（数据源）、Tools（可执行操作）、Prompts（模板提示） (3) MCP 在 Web3 场景下的应用——链上数据接入、智能合约工具调用
- 💡 产品思考：如果你设计一个 AI × Web3 产品，哪些能力应该通过 MCP Server 暴露？哪些应该内置？
