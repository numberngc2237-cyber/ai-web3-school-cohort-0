### 关键概念

1. **Agent 工作流 = 多步决策链路**：单个 Agent 调用一次工具就像"问一句答一句"——能查余额、能发交易，但不会自己判断下一步。工作流把它升级为"自动走完一条路"：查余额 → 判断够不够 → 发交易 → 等确认 → 通知用户。每一步的结果决定下一步走哪条分支。对无代码/产品方向的人来说，理解这一点就够了——工作流的本质是把「人类的一次指令」翻译成「Agent 的多步自动执行」。

2. **三种基本模式覆盖 80% 的场景**：顺序链（Chain）像流水线——A→B→C 一步步走；条件分支（Router）像岔路口——根据条件选不同路径；循环迭代（Loop）像打磨——重复执行直到满足条件。真实产品通常是三者的组合。比如链上定投：Router 判断"今天到定投日了吗？"→ Chain 执行"查余额→计算数量→下单"→ Loop "等链上确认，失败就重试"。

3. **Human-in-the-loop 是产品体验的分水岭**：全自动 Agent 爽但危险（一键买错币怎么办？），半自动 Agent 安全但繁琐。三种人机协同模式对应不同产品场景：审批（关键操作弹确认框）、修正（AI 先做，人检查后放行）、接管（AI 遇到不确定就停，等人来决策）。产品经理的核心判断：你的目标用户愿意用"安全感"换"自动化"吗？ 

### 产品视角

| 概念 | 产品场景 | 目标用户 | 解决什么问题 |
|------|----------|----------|-------------|
| 顺序链（Chain） | 一键领取多链空投：连接钱包 → 遍历链 → 查资格 → 逐个领取 → 汇总报告 | DeFi 重度用户 / 空投猎人 | 手动操作十几条链费时费力，容易漏领或被骗 |
| 条件分支（Router） | 智能止损机器人：监控价格 → 跌破阈值？→ 是：市价卖出 / 否：继续监控 | 合约交易者 / 无法盯盘的散户 | 情绪化交易和盯盘焦虑——机器按规则执行 |
| 循环迭代（Loop） | Gas 最优交易执行器：提交交易 → Gas 太高？→ 等待 → 重新估价 → 提交 | 所有链上交易用户 | Gas 波动时避免多付钱，适合大额交易 |
| Human-in-the-loop | AI 投顾：AI 生成投资建议 → 推送到用户手机 → 用户确认后 AI 执行 | 新手用户 / 保守型投资者 | "AI 帮我做，但我来拍板"——降低门槛但不丧失控制感 |

> 💡 产品洞察：Web3 Agent 工作流和传统 SaaS 工作流（如 Zapier）最大的不同是**交易不可逆 + Gas 成本**。Zapier 失败大不了重跑一次，链上失败的代价是真金白银。这意味着 Web3 Agent 工作流产品必须在"鲁棒性"上多一层设计——重试策略、Gas 估算、失败回滚。

## 问题和卡点
