## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ 核心概念。理解：什么是 Agent 工作流？为什么需要工作流编排？

- 📖 [智能体工作流 Agent Workflow](https://aiweb3.school/zh/handbook/bridge/agent-workflow/) — 为什么要学 + 第一性原理
- 🎯 重点理解三个问题：(1) Agent 工作流编排解决了什么核心问题？——单个工具调用 → 多步决策链路的跃迁 (2) 工作流的三种基本模式：顺序链（Chain）、条件分支（Router）、循环迭代（Loop）(3) 人工智能 vs 人工干预——哪些环节必须保留用户确认？
- 💡 产品思考：如果做一个「AI 链上自动定投 Agent」，从用户说「每周定投 100U ETH」到实际执行，中间需要经过哪些步骤？哪些步骤可以自动化、哪些必须人工确认？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读「工作流模式」+「Agent 权限」+「在 AI × Web3 中的位置」。

- 📖 同上链接 — 工作流模式详解 + Agent 权限设计 + AI × Web3 中的位置
- 🎯 重点了解：(1) DAG（有向无环图）工作流——多 Agent 协作的编排方式 (2) 人机协同模式——Human-in-the-loop 的三种实现：审批、修正、接管 (3) Agent 工作流在 Web3 场景的特殊约束——交易确定性、Gas 成本、MEV 保护
- 💡 产品思考：对比「全自动 Agent」和「半自动 Agent」两种产品形态，各适合什么用户画像？谁更愿意为哪种产品付费？

### 🟣 挑战路径（~2h）

完成推荐路径后，做产品分析 + 阅读扩展资料。

- 📖 同上链接 — 完整阅读 + 扩展阅读
- ✍️ 产品分析作业：选一个链上 AI Agent 项目（如 Virtuals Protocol / aixbt / Truth Terminal），分析它的 Agent 工作流设计——输入是什么？经过哪些步骤？输出是什么？人工干预点在哪里？尝试画一张流程图。
- 📚 扩展阅读：[LangGraph 文档](https://langchain-ai.github.io/langgraph/)、[CrewAI 多 Agent 工作流](https://docs.crewai.com/)、[ElizaOS 工作流引擎](https://elizaos.github.io/eliza/)
- 🔬 产品体验（可选）：试用一个支持 Agent 工作流的平台（如 Dify 的工作流模式），感受可视化编排的体验
