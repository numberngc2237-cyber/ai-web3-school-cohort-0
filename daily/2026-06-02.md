## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ 核心概念。理解：什么是 AI 框架？为什么要用框架而不是直接调 API？

- 📖 [框架 Frameworks](https://aiweb3.school/zh/handbook/ai/frameworks/) — 为什么要学 + 第一性原理
- 🎯 重点理解三个问题：(1) 框架解决了什么痛点？——从裸调 API 到结构化应用开发的演进 (2) 框架的核心能力：提示管理、链式调用、工具集成、记忆管理 (3) 为什么产品经理需要了解框架？——选错框架=选错技术路线，影响产品形态和迭代速度
- 💡 产品思考：如果你要做一个 AI Agent 产品，你会先选框架再设计产品，还是先设计产品再选框架？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读「主流框架对比」+「选型指南」+「AI × Web3 中的位置」。

- 📖 同上链接 — LangChain / LangGraph / CrewAI / Hermes 对比 + 选型指南 + AI × Web3 中的位置
- 🎯 重点了解：(1) LangChain：通用 LLM 应用框架，链式调用，生态最大 (2) LangGraph：状态图编排，适合多步 Agent 工作流 (3) CrewAI：多 Agent 协作框架，角色分工 (4) Hermes：AI × Web3 专用，链上工具原生集成
- 💡 产品思考：这四种框架分别对应的典型产品形态是什么？各适合什么规模的团队？

### 🟣 挑战路径（~2h）

完成推荐路径后，做框架对比表格 + 阅读扩展资料。

- 📖 同上链接 — 完整阅读全章 + 扩展阅读
- ✍️ 产品分析作业：画一张「AI 框架产品矩阵」——横轴「开发门槛（低→高）」，纵轴「适用场景（通用→Web3专用）」，把 LangChain / LangGraph / CrewAI / Hermes / Vibe Coding 工具放进去
- 📚 扩展阅读：[LangChain 文档](https://docs.langchain.com/)、[LangGraph 文档](https://langchain-ai.github.io/langgraph/)、[CrewAI 文档](https://docs.crewai.com/)、[Hermes Agent](https://hermes-agent.nousresearch.com/docs/)
- 🔬 产品体验（可选）：打开 [LangChain Academy](https://academy.langchain.com/) 浏览 Introduction 模块，感受一下框架的「入门门槛」
