## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ On-chain Data + Explorer Context + Citation。

- 📖 [链感知上下文 Chain-aware Context](https://aiweb3.school/zh/handbook/bridge/chain-aware-context/) — 为什么要学 + 第一性原理
- 🎯 重点理解：为什么 AI 不能凭记忆判断链上事实？链上状态有时间性——同一个地址的余额、授权和仓位在不断变化，Agent 必须实时读取而非凭历史回答。三个核心原则：(1) 链上状态有时间性 (2) 上下文要带来源 (3) 区分事实和解释
- 💡 产品思考：如果一个 DeFi 产品内置了 AI 顾问，不给它实时链感知上下文和给了它的回答会有什么不同？用户看到"可能是这样"vs"你当前余额 0.5 ETH"两种回答，信任感差异有多大？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读 Contract Docs + ABI/Event + Transaction History +「在 AI × Web3 中的位置」。

- 📖 同上链接 — Contract Docs + ABI/Event + Transaction History + 在 AI × Web3 中的位置
- 🎯 重点理解：ABI 让工具函数编码调用，但仅凭 ABI 不知道参数含义——需要合约文档（NatSpec）补充业务语义。Transaction History 不是"查交易记录"那么简单——它帮助 Agent 判断用户是否已授权、策略是否执行过、合约是否升级过、是否与高风险地址交互过。
- 💡 产品思考：一个好的链感知上下文包应该包含哪些字段？如果让你给 AI Agent 设计一个"标准化上下文接口"，你会要求哪些必需字段、哪些可选字段？

### 🟣 挑战路径（~2h）

完成推荐路径后，读 Indexing Context + 做「最小实践」+ 阅读扩展资料。

- 📖 同上链接 — Indexing Context + 最小实践
- 🔬 最小实践：去 Etherscan 找一笔公开交易 → 收集 chain id、block number、from、to、method、value、token transfers、logs → 找到合约 ABI 或 verified source → 写一段**模型可读的上下文**，每个关键结论附 tx hash 或 explorer 链接 → **标出哪些是链上事实、哪些是你的解释**
- 📚 扩展阅读：[Ethereum JSON-RPC](https://ethereum.org/en/developers/docs/apis/json-rpc/)、[The Graph Subgraphs](https://thegraph.com/docs/en/subgraphs/overview/)、[Etherscan API](https://docs.etherscan.io/)
- ✍️ 产品分析：在 AI × Web3 Bridge 的 15 个章节中，链感知上下文为什么被放在第一篇？如果跳过这章直接学 Web3 Tool Use 会有什么风险？
