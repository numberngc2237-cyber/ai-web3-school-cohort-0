# 2026-05-28 学习笔记

## 今日主题

**🚀 Phase 2：Web3 基础 — Day 5**：账户抽象（Account Abstraction）

> 昨天完成了智能合约（2.4）——理解了合约是把"信任"从人转移到代码，但部署后修改代价极大。今天进入账户抽象（2.5），学习如何把账户控制权从单一私钥扩展成**可编程规则**——这是 AI Agent 能自主执行链上操作的关键基础。

## 今日学习路径

> 根据你的画像（AI新手 / Web3新手 / 无代码 / 产品研究方向 / 每日2h），选一条开始：

### 🟢 最小路径（~30min）

先读「为什么要学」+「第一性原理」+ EOA 痛点 + Smart Account 初级知识节点。

- 📖 [账户抽象 Account Abstraction](https://aiweb3.school/zh/handbook/web3/account-abstraction/) — 为什么要学 + 第一性原理 + ERC-4337 + Smart Account
- 🎯 重点理解：传统 EOA 为什么"反人性"（私钥丢了=全丢、每步都要签名、必须持有 ETH 才能操作）；Smart Account 如何把权限从"有私钥/没私钥"变成"在什么条件下允许什么动作"
- 💡 产品思考：如果一个新产品要让 Web2 用户零门槛进入 Web3，账户抽象能解决哪些 onboarding 摩擦？

### 🔵 推荐路径（~1h）

完成最小路径后，继续读 Bundler + Paymaster +「在 AI × Web3 中的位置」。

- 📖 同上链接 — Bundler + Paymaster + AI × Web3 中的位置
- 🎯 重点理解：Bundler 是基础设施——它不稳定，用户操作就会卡住；Paymaster 可以让新用户没 ETH 也能操作（但需要风控）；账户抽象是 AI Agent 上链执行的底座
- 💡 产品思考：如果设计一个 Paymaster 策略，如何平衡用户体验（免 gas）和防滥用（防 spam / 套利）？

### 🟣 挑战路径（~2h）

完成推荐路径后，读 Session Key（高级）+ 做「最小实践」+ 阅读扩展资料。

- 📖 同上链接 — Session Key + 最小实践
- 🔬 最小实践：选一个场景（如"每小时最多再平衡一次测试网小额资产"），设计一个 Agent Session Key 策略——允许调用的合约和方法、额度上限、过期时间、哪些动作必须回到用户钱包确认
- 📚 扩展阅读：[EIP-4337](https://eips.ethereum.org/EIPS/eip-4337)，[Safe Core](https://safe.global/core)，[Rhinestone](https://www.rhinestone.wtf/)
- ✍️ 产品分析草稿：对比 Safe、Argent、Biconomy 三个 Smart Account 产品的差异——它们在"权限可编程"这个维度上各自做了什么取舍？
