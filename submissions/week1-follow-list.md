## 我为什么这样选择（按类别说明）

### AI 方向（Z.AI、GoPlus、Phala、DeepFunding）

我在追踪 AI Agent 的产品化基础设施。Z.AI 是模型和工具层，GoPlus 是安全层，Phala 是可信执行层，DeepFunding 是治理层——这四个拼在一起刚好构成 Agent 产品的完整技术栈。我的学习目标不是写代码，而是理解每一层的产品价值、用户场景和商业逻辑。

### Web3 方向（ETHPanda、Vitalik、Bruce、LXDAO）

我的 Web3 基础是新手，需要建立底层认知。Vitalik 提供长期路线图视角，ETHPanda 和 Bruce 提供华语生态的具体实践，LXDAO 让我观察开源项目怎么走向产品化。

### AI × Web3 交叉（Elytro、AI Web3 School、WCB）

这是我 Track 的核心——Agent Wallet 的产品边界在哪？什么自动、什么人工确认？Elytro 是目前少数在探索这个方向的产品，AI Web3 School 和 WCB 是我的学习锚点。

### 安全（GoPlus、Phala）

安全不是独立方向，而是 Agent 产品必须内置的能力。GoPlus 做执行前检测，Phala 做可信执行——两者组合是「可验证 Agent 身份」的关键基础设施。我关注它们不是研究安全技术本身，而是研究「Agent 安全」怎么成为产品的差异化卖点。

### 课程生态（AI Web3 School、ETHPanda、LXDAO、WCB）

这四个是我的学习坐标系——知道课程在推进什么、社区在讨论什么、其他学员在做什么。

## 关于我还不理解的问题

作为 Web3 新手，我目前在信息流中遇到的、还不理解的术语和概念。我不会急着现在全部弄懂，而是带着它们在信息流中持续观察：

| 术语 | 我目前的理解 | 我在观察什么 |
|------|------------|------------|
| MPC | 多方计算，钱包不需要一个人保管私钥 | 产品怎么向用户解释「安全」？用户需要知道 MPC 这个词吗？ |
| TEE | 可信执行环境，代码在隔离区运行 | 和 MPC 是什么关系？用户选择产品时需要考虑 TEE 吗？ |
| Session Keys | 让 Agent 替我做有限的事 | 用户体验长什么样？权限过期了怎么办？ |
| Intent-based | 用户说意图，系统找最优路径执行 | 这和传统支付体验的差别在哪？用户感知得到吗？ |
| ERC-4337 | 账户抽象，钱包可以自定义规则 | 这能催生出什么新产品？对普通用户有什么价值？ |
| Agent Wallet | AI 有自己的钱包 | 用户怎么控制、怎么撤销、怎么审计？ |
